Comandos do GIT

git init

Realiza inicialização do repositório na pasta atual. Por enquanto isso fica somente na máquina do usuario ainda.

git status

Ve o estado atual de todos os arquivos.

git add "nome do arquivo" ou " . "

Adiciona os arquivos no fluxo para versionamento podendo mandar individualmente cada arquivo ou usar um ponto para mandar todos de uma vez.

Com os comandos acimas os arquivos estão no "palco" ainda, na area de change, eles estão sendo trackeados mas ainda não foi commitados.

git commit -a -m " mensagem do commit "

Neste comando temos dois parametros o -a é para adicionar todos os tipos de arquivos neste commit.

O parametro -m é para colocar a mensagem do commit tipo o que foi implementado ou alterado

Após este comando os arquivos estão de fato no GIT ainda no computador na pastado projeto.

A partir de agora vamos disponibilizar para o repositório remoto. Primeiro deve ser criado no GitHUb o repositório, esta parte não sera descrita aqui. Após a criação la dara os comandos também, abaixo veremos a estrutura.

git remote add origin https://github.com/usuario/repositório

Faz a conexão dos arquivos que estão no GIT com o repositório no GITHUB. Observe que no lugar do usuario sera a sua conta e no repositório o nome que voce colocou.

git push -u origin main

Envia os arquivos para o repositório remoto, salvando eles no GITHUB.

Agora veremos uma parte importantissima que são as Branchs, isso basicamente podemos dizer que são ramificações. Vamos entender com um exemplo:

Imagine que tem um projeto que 3 pessoas ira trabalhar nele, ja tem a estrutura um vai fazer o header da pagina outro um formulario de contato e outro o footer.


Então pode ser criados as branchs e cada um trabalha na sua e depois pode juntar tudo na branch main.

git checkout -b " nome da branch "

O -b indica a criação de uma nova branch, importante antes de criar uma branch estar com os commits anteriores feitos

git push --set-upstream origin " nome da branch "

Este comando cria a branch no repositório remoto, porque se tentar fazer push antes dara erro, porque não existe a branch remotamente ainda.


Agora pode alterar os arquivos do projeto realizar os commit e o push onde sera enviado para a branch as modificações.

Agora veremos os comando para unir a branch criada com a main unindo os arquivos

git checkout main

Agoara nao sera criado uma branch nova, só sera mudado de branch, saindo da que estava para a raiz

git brach

Ve a lista com todas as branchs que tem no projeto com os nomes por isso tem de colocar nomes que faz sentido ficando facil reconhecer.

git merge " nome da branch "

Vai pegar o código da branch e disponibilar na main as alterações. Ainda precisa passar isso para o repositorio porque até então só esta na maquina este merge.

git push

Envia para o repositório remoto a branch que foi mergeada na main, deixou os codigos iguais na main e na branch

Pegar um projeto d eum repositório para trabalhar nele.

git pull

Pega todas as atualizações do remoto e traz para repositório local.

Clonar um projeto de um repositório. Va no projeto e pega a URL na aba code.

git clone " URL copiada " nome do projeto

Após isso o projeto vai estar na sua máquina vindo da branch mainonde pode alterar ai depois faz o git commit -a -m "mensagem" e depois git push.
